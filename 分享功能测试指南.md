# 分享功能测试指南

## 功能概述

本次实现的分享功能包含两个主要特性：
1. **转发为话题**：用户可以将感兴趣的话题转发并发布为新话题，系统自动添加"转发"二级标签和"分享"四级标签
2. **生成分享链接**：用户可以生成话题的分享链接并复制到剪贴板

## 后端实现

### 新增接口

1. **生成分享链接**
   - 接口：`POST /api/topics/{id}/share-url`
   - 方法：`TopicsController.generateShareUrl()`
   - 返回：包含分享链接、标题、描述等信息

2. **获取分享信息**
   - 接口：`GET /api/topics/{id}/share-info`
   - 方法：`TopicsController.getShareInfo()`
   - 返回：话题的基本信息用于分享展示

### 服务层实现

- `TopicsService.generateShareUrl()` - 生成分享链接
- `TopicsService.getShareInfo()` - 获取分享信息

## 前端实现

### 组件更新

1. **TopicDetail.vue**
   - 优化分享按钮点击事件处理
   - 实现两种分享方式的选择界面
   - 调用后端接口生成分享链接

2. **PublishTopicModal.vue**
   - 监听分享数据并自动填充内容
   - 自动添加"转发"二级标签（代码：forward）
   - 自动添加"分享"四级标签（代码：share）
   - 显示分享信息来源提示

3. **Topicwall.vue**
   - 处理 URL 参数中的分享来源标识
   - 自动打开分享弹窗

### API 更新

在 `src/api/topic.js` 中新增：
- `topicAPI.generateShareUrl(topicId)` - 生成分享链接
- `topicAPI.getShareInfo(topicId)` - 获取分享信息

## 测试步骤

### 前置条件

1. 确保后端服务已启动
2. 确保前端开发服务器已启动
3. 执行数据库 SQL 脚本添加必要标签
4. 使用有效账号登录

### 测试场景 1：转发为话题

1. 进入话题详情页（`/topic/{id}`）
2. 点击"分享"按钮
3. 选择"📝 转发为话题"选项
4. 系统应跳转到话题墙页面并自动打开发布弹窗
5. 检查以下内容：
   - ✅ 内容框自动填充分享格式的内容
   - ✅ 显示分享信息来源提示框
   - ✅ 二级标签自动选中"转发"
   - ✅ 四级标签自动选中"分享"
   - ✅ 可以补充说明文字
   - ✅ 发布成功后能在话题墙看到新话题

### 测试场景 2：生成分享链接

1. 进入话题详情页
2. 点击"分享"按钮
3. 选择"🔗 生成分享链接"选项
4. 检查以下内容：
   - ✅ 调用后端接口生成分享链接
   - ✅ 链接自动复制到剪贴板
   - ✅ 显示成功提示信息
   - ✅ 可以在浏览器地址栏粘贴链接访问

### 测试场景 3：直接访问带分享参数的 URL

1. 访问：`http://localhost:5173/topicwall?from=share&sourceType=topic&sourceId=1`
2. 检查以下内容：
   - ✅ 页面加载后自动打开发布弹窗
   - ✅ 分享内容正确填充
   - ✅ 标签自动选择正确

## 预期结果

### 转发内容格式
```
【分享自话题】@用户名：原话题内容前 100 字符...

原文链接：http://domain/topic/话题 ID
```

### 自动标签
- 一级标签：根据用户身份自动选择（学生/商户/团体等）
- 二级标签：自动添加"转发"（代码：forward）
- 四级标签：自动添加"分享"（代码：share）

## 错误处理

1. **话题不存在**
   - 前端应显示"话题不存在"错误提示
   
2. **网络错误**
   - 后端接口失败时应降级到本地生成链接
   - 显示友好的错误提示

3. **标签不存在**
   - 如果"转发"或"分享"标签不存在，发布时应提示用户手动选择标签
