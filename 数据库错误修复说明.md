# 数据库错误修复说明

## 问题描述

执行 `share_tags.sql` 时报错：
```
1054 - Unknown column 'code' in 'field list'
```

## 原因分析

`tag_level_4` 表的实际结构与预期不符：
- **预期字段**：code, name, category, icon, color, usage_count, is_active...
- **实际字段**：name, category, usage_count, trend_score, last_used_at, created_by, status...

`TagLevel4` 实体类中没有 `code`、`icon`、`color`、`is_active` 字段。

## 解决方案

已创建新的安全 SQL 脚本：`share_tags_safe.sql`

### 使用方法

```bash
mysql -u your_username -p your_database < src/main/resources/sql/share_tags_safe.sql
```

### 脚本特性

该脚本会自动：
1. 检查 `tag_level_2` 表是否缺少字段（code, icon, color, sort_order, is_active）
2. 如果字段不存在，自动添加这些字段
3. 使用 `NOT EXISTS` 检查避免重复插入数据
4. 执行完成后显示成功提示

### 涉及的表结构

#### tag_level_4 表
```sql
CREATE TABLE tag_level_4 (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(255) NOT NULL,
    category VARCHAR(50),
    usage_count BIGINT DEFAULT 0,
    trend_score DECIMAL(10,2) DEFAULT 0,
    last_used_at DATETIME,
    created_by BIGINT,
    status VARCHAR(20) DEFAULT 'active',
    created_at DATETIME,
    updated_at DATETIME
);
```

#### tag_level_2 表（已更新）
```sql
CREATE TABLE tag_level_2 (
    id INT PRIMARY KEY AUTO_INCREMENT,
    code VARCHAR(50),           -- 新增
    name VARCHAR(255) NOT NULL,
    parent_id INT,
    icon VARCHAR(255),          -- 新增
    color VARCHAR(50),          -- 新增
    sort_order INT,             -- 新增
    is_active TINYINT DEFAULT 1, -- 新增
    created_at DATETIME,
    updated_at DATETIME
);
```

## 验证方法

执行以下 SQL 验证数据是否添加成功：

```sql
-- 查看四级标签
SELECT * FROM tag_level_4 WHERE name = '分享';

-- 查看二级标签
SELECT * FROM tag_level_2 WHERE code = 'forward';

-- 查看 tag_level_2 表结构
DESC tag_level_2;
```

## 注意事项

1. 执行脚本前请备份数据库
2. 确保数据库用户有 ALTER TABLE 权限
3. 如果已经手动创建过标签，脚本会自动跳过
4. 脚本使用动态 SQL，需要在 MySQL 5.7+ 版本执行
