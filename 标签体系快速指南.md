## 📊 标签体系总览

### 5 级标签说明

| 级别 | 表名 | 用途 | 数据量 | 示例 |
|------|------|------|--------|------|
| Level 1 | tag_level_1 | 身份标签 | 6 条 | 学生、商户、管理员 |
| Level 2 | tag_level_2 | 话题分类 | 9 条 | 学习经验、技术交流 |
| Level 3 | tag_level_3 | 地点标签 | 6 条 | 图书馆、食堂、宿舍 |
| Level 4 | tag_level_4 | 自定义话题 | 0 条 | 用户创建的新标签 |
| Level 5 | tag_level_5 | 商品标签 | 7 条 | 二手物品、教材书籍 |

### 新三分法标签

| 表名 | 用途 | 数据来源 | 数据量 |
|------|------|---------|--------|
| topic_tag | 话题标签 | tag_level_1 + tag_level_2 | 9 条 |
| product_tag | 商品标签 | tag_level_5 | 7 条 |
| location_tag | 地点标签 | tag_level_3 | 6 条 |

## 🎯 前端使用场景

### 1. 发布话题

**组件**: `TagSelector.vue`

**标签选择规则**:
- **身份标签 (Level 1)**: 单选，自动根据用户身份选择
- **话题分类 (Level 2)**: 多选，最多 3 个
- **地点标签 (Level 3)**: 可选，多选，最多 2 个
- **自定义标签 (Level 4)**: 可选，多选，最多 5 个

**示例代码**:
```vue
<tag-selector 
  v-model="selectedTags"
  :show-level1="true"
  :show-level2="true"
  :show-level3="true"
  :show-level4="true"
  :auto-select-level1="true"
/>
```

### 2. 发布商品

**组件**: `PublishProduct.vue`

**标签选择**:
- 点击"选择/自定义标签"按钮
- 从商品标签池选择（二级标签）
- 支持创建自定义标签

**数据结构**:
```json
{
  "productTags": [
    {"name": "secondhand", "color": "#4A90E2"},
    {"name": "books", "color": "#50C878"}
  ]
}
```

### 3. 话题墙筛选

**组件**: `Topicwall.vue` + `TopicFilter.vue`

**筛选维度**:
- 一级标签（身份）：全部、学生、团体、关注、社会
- 二级标签（话题）：学习经验、技术交流等 8 个
- 三级标签（地点）：图书馆、食堂等 6 个

## 📝 数据库操作

### 查看标签数据

```sql
-- 查看所有身份标签
SELECT * FROM tag_level_1 ORDER BY sort_order;

-- 查看所有话题分类标签
SELECT * FROM tag_level_2 ORDER BY sort_order;

-- 查看所有地点标签
SELECT * FROM tag_level_3 ORDER BY sort_order;

-- 查看所有商品标签
SELECT * FROM tag_level_5 ORDER BY sort_order;

-- 查看热门话题标签（按趋势分数排序）
SELECT * FROM topic_tag 
WHERE is_active = 1 AND status = 'active'
ORDER BY trend_score DESC 
LIMIT 10;
```

### 增加标签使用次数

```java
// Java 代码示例
@Autowired
private TopicTagService topicTagService;

// 当用户使用某个标签时
List<String> tagCodes = Arrays.asList("study", "tech_exchange");
topicTagService.increaseUsage(tagCodes);
```

### 热门标签算法

```java
// 趋势分数 = 使用次数 × 0.1
tag.setTrendScore(tag.getUsageCount() * 0.1);

// 按趋势分数排序获取热门标签
QueryWrapper<TopicTag> wrapper = new QueryWrapper<>();
wrapper.eq("is_active", true)
       .and(w -> w.eq("type", "system").or().eq("status", "active"));
wrapper.orderByDesc("trend_score");
wrapper.last("LIMIT 10");

List<TopicTag> hotTags = topicTagService.list(wrapper);
```

## ⚠️ 常见问题

### Q1: 为什么前端显示硬编码的标签？

**A**: Topicwall.vue 中使用了硬编码的 displayLevel1Tags 和 displayLevel2Tags。

**解决方案**: 
```javascript
// 改为从后端获取
const level1Tags = ref([]);
const level2Tags = ref([]);

onMounted(async () => {
  level1Tags.value = await api.get('/api/tags/level1');
  level2Tags.value = await api.get('/api/tags/level2');
});
```

### Q2: 如何添加新的标签？

**A**: 两种方式：

1. **管理员后台添加**:
   ```
   访问管理后台 → 标签管理 → 添加标签
   ```

2. **直接插入数据库**:
   ```sql
   INSERT INTO tag_level_2 (code, name, icon, color, sort_order, is_active)
   VALUES ('new_tag', '新标签', '🎯', '#FF0000', 100, 1);
   ```

### Q3: 用户自定义标签如何使用？

**A**: 
1. 用户发布话题时创建自定义标签
2. 标签状态为 `pending`（待审核）
3. 管理员审核后状态变为 `active`
4. 其他用户可以在标签池看到并使用

### Q4: 标签可以删除吗？

**A**: 
- **系统标签**: 不建议删除，可以设置 `is_active = 0` 禁用
- **用户自定义标签**: 可以删除，但需要先检查是否被使用

```sql
-- 软删除（推荐）
UPDATE tag_level_2 SET is_active = 0 WHERE id = 123;

-- 硬删除（谨慎）
DELETE FROM tag_level_2 WHERE id = 123;
```

```sql
-- 查看标签关联的话题数
SELECT 
  t.name,
  COUNT(DISTINCT tp.id) AS topic_count
FROM topic_tag t
LEFT JOIN topics tp ON JSON_CONTAINS(
  JSON_EXTRACT(tp.topic_tag_codes, '$'), 
  CONCAT('"', t.name, '"')
)
GROUP BY t.id, t.name
ORDER BY topic_count DESC;
```
