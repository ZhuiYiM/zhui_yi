## 二、标签体系架构

### 2.1 设计理念

项目采用**双轨制标签体系**：

#### 原设计（五级标签）
```
tag_level_1 → 身份标签（学生、商户、管理员...）
tag_level_2 → 话题分类（学习经验、技术交流...）
tag_level_3 → 地点标签（图书馆、食堂...）
tag_level_4 → 自定义话题（用户创建）
tag_level_5 → 商业标签（二手、书籍、数码...）
```

#### 新设计（三分法标签）
```
topic_tag    → 话题相关（身份 + 话题分类）
product_tag  → 商品相关（商品分类）
location_tag → 地点相关（位置标记）
```

### 2.2 对应关系

| 原标签表 | 新标签表 | 说明 |
|---------|---------|------|
| tag_level_1 (身份) | topic_tag (部分) | 身份标签合并到话题标签 |
| tag_level_2 (话题) | topic_tag (部分) | 话题分类合并到话题标签 |
| tag_level_3 (地点) | location_tag | 地点标签独立 |
| tag_level_4 (自定义) | 保持 tag_level_4 | 保留用于用户自定义 |
| tag_level_5 (商品) | product_tag | 商品标签独立 |

## 四、API 接口清单

### 4.1 公共标签 API (`/api/tags`)

```
GET /api/tags/topics          → 获取所有话题标签
GET /api/tags/level1          → 获取身份标签
GET /api/tags/level2          → 获取二级标签
GET /api/tags/level3          → 获取三级标签
GET /api/tags/level4          → 获取四级标签（分页）
GET /api/tags/level5          → 获取五级标签

GET /api/tags/topics/hot      → 获取热门话题标签
GET /api/tags/products/hot    → 获取热门商品标签
GET /api/tags/locations/hot   → 获取热门地点标签

POST /api/tags/topics/custom  → 创建自定义话题标签
POST /api/tags/products/custom → 创建自定义商品标签
POST /api/tags/locations/custom → 创建自定义地点标签

POST /api/tags/{type}/{id}/audit    → 审核标签
POST /api/tags/{type}/{id}/convert  → 转为系统标签
```

### 4.2 管理员标签 API (`/api/admin/tags`)

```
GET  /admin/tags/level-2           → 分页查询二级标签
POST /admin/tags/level-2           → 创建二级标签
PUT  /admin/tags/level-2/{id}      → 更新二级标签
DELETE /admin/tags/level-2/{id}    → 删除二级标签
DELETE /admin/tags/level-2/batch   → 批量删除

// 类似接口 for level 3, 4, 5
```

## 六、发现的问题

### 6.1 数据库层面

1. **缺失 tag_level_X 表**
   - 后端代码有实体类，但数据库表不存在
   - 导致 TagService 无法正常工作
   - 需要执行 SQL 脚本创建表并迁移数据

2. **数据不完整**
   - location_tag 缺少 icon 和 color
   - topic_tag 中 `studys` 可能为拼写错误
   - 部分标签 icon 字段为 emoji 乱码

3. **历史遗留问题**
   - `tags` 表是旧标签表，应该考虑迁移或删除

### 6.2 后端层面

1. **服务层混乱**
   - 同时存在 TagService 和 TopicTagService 等
   - 职责划分不清晰
   - 可能导致重复代码

2. **缺少数据验证**
   - 创建标签时没有名称唯一性校验
   - 缺少标签使用统计的详细记录

### 6.3 前端层面

1. **硬编码问题**
   - Topicwall.vue 中 displayLevel1Tags 等使用硬编码
   - 应该优先使用后端返回的数据

2. **缺少错误处理**
   - 标签加载失败时没有友好的错误提示
   - 网络异常处理不完善
