## ✅ 修复方案

### 1. 前端修复（已完成）

**文件**: [`PublishTopicModal.vue`](file:///F:/code/CampusInformationPlatform/demo/src/components/topic/PublishTopicModal.vue)

**修复内容**:

```javascript
// 构建请求数据
const topicData = {
  content: localContent.value,
  images: imageUrls,
  level1TagCode: selectedTagsData.level1?.code || null,
  // 话题标签（level2）
  topicTagCodes: (selectedTagsData.level2 || []).map(t => t.code),
  // 地点标签（level3）
  locationTagCodes: (selectedTagsData.level3 || []).map(t => t.code),
  // 自定义标签（level4）需要合并到 topicTagCodes 中
  // 因为后端将自定义话题标签也存储在 topic_tag_codes 字段中
  anonymous: false
};

// 🔧 关键修复：将 level4 自定义标签合并到 topicTagCodes
if (selectedTagsData.level4 && selectedTagsData.level4.length > 0) {
  const level4TagNames = selectedTagsData.level4.map(t => t.name);
  // 将 level4 标签名称添加到 topicTagCodes 中
  topicData.topicTagCodes = [
    ...(selectedTagsData.level2 || []).map(t => t.code),
    ...level4TagNames
  ];
  console.log('🏷️ 合并后的话题标签:', topicData.topicTagCodes);
}
```

**修复原理**:
- 将 `level2`（系统话题标签）的 code 和 `level4`（自定义标签）的 name 合并到 `topicTagCodes` 数组
- 这样后端可以将所有话题相关的标签统一存储到 `topic_tag_codes` 字段
- 保持了向后兼容性，不影响现有功能
